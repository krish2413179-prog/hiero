# Contributing to Hiero TS Utilities

We welcome contributions! To ensure high quality and clear legal provenance, we use the Developer Certificate of Origin (DCO) and require GPG-signed commits for certain integrations.

## Developer Certificate of Origin (DCO)

By contributing to this project, you certify that you have the right to submit the work and agree that the contribution may be licensed under Apache-2.0.

Please sign off on your commits by adding `Signed-off-by: Name <email>` to the end of your commit messages. This can be done automatically using `git commit -s`.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Implement your changes** and include tests.
3. **Ensure tests pass** by running `npm run test`.
4. **Submit a Pull Request** describing the changes and why they are needed.

## Code Standards

- Follow the existing coding style (Prettier/ESLint).
- Use descriptive variable and function names.
- Document public APIs with JSDoc.
- Keep components modular and reusable.
